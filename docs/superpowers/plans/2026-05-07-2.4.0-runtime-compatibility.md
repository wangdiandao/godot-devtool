# 2.4.0 Runtime Compatibility Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `godot-devtool` 2.4.0 with executable runtime input and recording compatibility routes plus synchronized metadata and bilingual docs.

**Architecture:** Keep runtime command behavior in the Godot addon/router and mirror it in the installed runtime bridge generator. Improve TypeScript tool-definition metadata so `get_capabilities` and README generation describe real runtime routes accurately.

**Tech Stack:** TypeScript ES modules, Node assertion verification scripts, GDScript runtime bridge/addon source, generated Markdown README tables.

---

## File Structure

- Modify `scripts/verify-godot-runtime.js` to add failing checks for source addon input injection, source addon recording routes, installed bridge recording capture, and sequence failure behavior.
- Modify `src/addons/godot_devtool/commands/runtime_commands.gd` to implement runtime input simulation and recording routes in the source addon.
- Modify `src/godot/editorBridge.ts` to align generated installed runtime bridge validation and sequence/replay behavior with the source addon.
- Modify `src/tools/compatibilityTools.ts` and `src/tools/definitions/compatibility.ts` to mark runtime exact-name wrappers as bridge-backed runtime routes.
- Modify `scripts/generate-readme-tool-tables.js`, `README.md`, and `README.zh-CN.md` through the existing generation flow.
- Modify `package.json`, `package-lock.json`, `skills/godot-devtool/SKILL.md`, `CHANGELOG.md`, `CHANGELOG.zh-CN.md`, `ROADMAP.md`, `ROADMAP.zh-CN.md`, and verification scripts for `2.4.0` synchronization.
