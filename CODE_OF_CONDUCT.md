# Code of Conduct

[中文](CODE_OF_CONDUCT.zh-CN.md) | English

## Purpose

`godot-devtool` is built for AI-assisted Godot Engine development workflows. The project welcomes constructive participation from users, contributors, maintainers, and downstream package distributors.

This Code of Conduct sets the baseline for respectful collaboration in project repositories, issue trackers, pull requests, discussions, release channels, and other official project spaces.

## Expected Behavior

Participants are expected to:

- Treat others with respect, patience, and professional courtesy.
- Focus technical disagreement on evidence, reproducible behavior, and project goals.
- Give and receive feedback in good faith.
- Respect differences in experience level, language, background, and viewpoint.
- Keep discussions relevant to the project and its users.
- Report security issues through appropriate private channels when public disclosure could harm users.

## Unacceptable Behavior

The following behavior is not acceptable in project spaces:

- Harassment, threats, intimidation, stalking, or sustained disruption.
- Insults, personal attacks, slurs, or discriminatory language.
- Sexualized language, imagery, or attention.
- Publishing private information without clear permission.
- Trolling, deliberate misinformation, or repeated off-topic escalation.
- Abusive pressure on maintainers, contributors, or users.
- Encouraging unsafe, unlawful, or clearly harmful use of the project.

## Maintainer Responsibilities

Maintainers may clarify expectations, moderate discussions, remove inappropriate content, restrict participation, or take other reasonable action to protect the project community.

Moderation decisions should be based on observed conduct, project safety, and the impact on affected participants. Maintainers should apply this Code of Conduct consistently and proportionally.

## Reporting

If you experience or observe unacceptable behavior, report it through the project's issue tracker or repository contact channels. If the matter involves a security concern, private information, or a sensitive personal issue, avoid posting details publicly and contact the maintainers privately when possible.

Reports should include:

- A description of the behavior.
- Links, screenshots, timestamps, or other relevant context when available.
- Whether immediate action is needed to reduce harm.

Maintainers should review reports promptly, respect privacy where practical, and communicate outcomes when appropriate.

## Scope

This Code of Conduct applies to official project spaces and to conduct outside those spaces when that conduct directly affects project participation, contributor safety, or user trust.

## Enforcement

Possible enforcement actions include:

- A private or public clarification of expected conduct.
- A warning.
- Temporary limits on participation.
- Removal of comments, issues, pull requests, or other content.
- Blocking, banning, or other repository-level restrictions.

Severe or repeated violations may lead to immediate restriction without prior warning.
